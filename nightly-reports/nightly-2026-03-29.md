## What Shipped Tonight

### The Catalog Is Live ‚Äî Full Asset Router Online

This is the big one. The asset catalog ‚Äî the backbone that lets mods seamlessly swap characters, textures, animations, and sounds without touching ROM data ‚Äî went from "plumbing in place" to "primary asset router" in a single marathon day. Sessions 74 through today's head commit pushed through C-1 all the way to C-9, plus the MEM-2 lifecycle API. Here's what that means in practice:

**C-1: ROM Hash Cache** ‚Äî SHA-256 integrity verification at startup. The game now fingerprints the ROM on first launch and caches the hash. If someone swaps the ROM out from under the game, it knows.

**C-2-ext through C-4: File Intercept** ‚Äî Every `fileLoadToNew` call in the engine now passes through `catalogGetFileOverride()` first. If a mod declares a replacement body file, the catalog intercepts the load transparently. Zero call-site changes needed ‚Äî all 16+ callers benefit for free.

**C-5/C-6/C-7: Texture, Animation, and Sound Intercepts** ‚Äî Same pattern, applied to `texLoad`, `animLoadFrame/Header`, and `sndStart`. Mods can now override any of the four core asset types. Basegame body/head fallback logic included so nothing breaks if a mod only partially overrides.

**C-8: Runtime Mod Toggle** ‚Äî Enabling or disabling a mod in the Modding Hub now triggers a full catalog rebuild. Override arrays are recomputed so the next stage load picks up the new mod state. No restart required.

**C-9/MEM-3: Stage Transition Diff** ‚Äî `catalogComputeStageDiff()` runs in `lvReset()` to detect what changed between the last stage and the next. This is the foundation for smart asset loading ‚Äî only load what's new, unload what's stale.

**MEM-2: Asset Lifecycle API** ‚Äî `catalogLoadAsset`, `catalogUnloadAsset`, `catalogRetainAsset`. The catalog can now track which assets are in memory, which are on disk, and which are held by active references. This closes the loop on memory modernization M0‚ÜíM2.

**CatalogResolveResult API** ‚Äî The final commit promoted the catalog to "primary asset router" status. All file loads now go through a unified resolve path that returns a typed result (catalog hit, ROM fallback, or error). The old spaghetti of mod path checks scattered across loading code is replaced by a single dispatcher.

**Capsule collision was disabled** ‚Äî The custom capsule collision system (D2b) was turned off and original collision restored. It wasn't ready for prime time, and rather than ship something half-baked that'd produce weird physics on certain geometry, the call was made to revert. The mesh code is preserved for a proper Phase 2 redesign. Right call ‚Äî shipping broken collision would've been worse than shipping the N64 original.

### Combat Sim ‚Äî Feels Like a Real Lobby Now

**Scenario Save/Load (S71)** ‚Äî You can now save your entire Combat Sim setup (arena, mode, limits, weapons, full bot roster) as a JSON file and reload it later. Want that 31-bot Ravine chaos config? Save it once, load it forever. Files live in `$S/scenarios/` and the UI has Save/Load buttons right in the Combat Sim tab. **Bot Name/Char Sync (S72, B-44)** ‚Äî CLC_LOBBY_START now carries per-bot name, bodynum, headnum, difficulty, and type. What you set up in the room is what spawns in the match. No more "why do all my bots look the same" moments.

**HUD Overlay (S69)** ‚Äî In-match HUD now shows the top 2 scorers and the match timer. You can actually tell who's winning without pausing.

**Player Identity Name Fix (B-26 re-fix)** ‚Äî The player name saga continues. The S49 identity profile fix relied on `identityGetActiveProfile()` which is null on clients. S72 rewired it to use `g_GameFile.name` as the fallback, which is the agent name from the save file. Should finally stick.

### Stability Fixes ‚Äî Clearing the Crash Backlog

**B-43: First-Tick Crash** ‚Äî NULL guard on `g_MpAllChrPtrs` during the very first tick of a match. The array wasn't populated yet but bot AI was already trying to read it. Classic race condition from the N64 days that never showed up with 4 players.

**B-45: Match-End Freeze** ‚Äî `MPPAUSEMODE_GAMEOVER` had no exit path on PC. The N64 used `prevmenuroot` to trigger the post-match menu, but ImGui doesn't observe that. Added a proper "MATCH OVER" screen with rankings and a "Return to Lobby" button.

**B-46: Void Spawn on MP Stages** ‚Äî Players were spawning at (0,0,0) and falling into the abyss on Felicity and other MP maps. Two causes: the intro validator was too aggressive (rejecting valid 1-word MP intros), and the B-19 pads fallback didn't cover local Combat Sim where netmode is NONE but `normmplayerisrunning` is true.

**B-47: Exit Freeze on Window Close** ‚Äî Alt-F4 during a match would hang the process for 10‚Äì30 seconds. `netDisconnect()` was trying to delete the UPnP port mapping (synchronous HTTP to router) and load a new stage (with no render loop running). Both paths now bail when `g_AppQuitting` is set. Port mappings expire naturally anyway.

**B-48: Arena Picker Garbled Names** ‚Äî GoldenEye X arenas were showing names like "Load A Saved Head" instead of "Frigate" because the AIO mod's language file maps those langids to old PerfectHead UI strings. Fixed by routing all arena name lookups through the existing `arenaGetName()` override table.

### Build Infrastructure

**Static Linking Complete** ‚Äî All dependencies are now statically linked. Zero runtime DLLs. The exe is self-contained. This took a few rounds of CMakeLists.txt recovery (there was a corruption incident, but it was resolved cleanly).

**Mod Reload Pipeline (T-8/T-9)** ‚Äî `stageTableReset()` truncates the stage table back to the base count on mod reload. `videoResetTextureCache()` flushes stale textures. `mainChangeToStage(TITLE)` forces a clean title screen load. Toggling mods on and off should now be fully clean with no ghost stages or stale textures.

**T-7: mod.json Content Registration** ‚Äî Mods can now declare bodies, heads, and arenas in their mod.json under a `content` block. The catalog picks them up at load time and they appear in the character/arena pickers.

### Bug Investigation ‚Äî B-49 (Instrumented, Not Yet Fixed)

**B-49: Felicity/Toilet Freeze** ‚Äî Game hangs after a long fall and landing. The freeze happens within the same tick as the landing sound creation, somewhere between `psCreate` (footstep sounds) and `func0f065e74` (room traversal). 15 checkpoint-style `JUMP_DEBUG:` log statements were added to `bwalkUpdateVertical()` to pinpoint the exact hang location on next reproduction. The toilet scenario at Carrington Institute is a separate but related case. This one's CRITICAL and still open.
